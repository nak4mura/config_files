# 業務要件抽出コマンド

現在のプロジェクトから業務要件を抽出し、機能タイプ別に整理して一覧化します。

## 抽出対象

- **ドキュメント**: README, docs/, requirements/, *.md
- **テストコード**: ユニットテスト、統合テスト、E2E テスト
- **API 仕様**: ルーティング定義、エンドポイント
- **DB スキーマ**: テーブル定義、制約、リレーション
- **コード内のドキュメント**: JSDoc, docstring, コメント
- **UI 定義**: フォームバリデーション、画面遷移
- **Issue/PR**: GitHub Issues, Pull Requests

## 出力内容

機能タイプ別に分類された業務要件一覧を生成します：

- 認証・認可
- データ入力・保存
- 検索・取得
- レポート・出力
- 通知
- 設定・管理
- データ同期
- その他

各要件には以下の情報を含みます：

- 要件の説明
- 詳細要件（制約、ルールなど）
- 出典（ファイルパス、行番号）
- 関連要素（API, DB, UI, Issue）

## 出力先

業務要件一覧は以下のパスに出力されます:
- `docs/requirements.md`

## 使用例

```bash
/extract-requirements
```

## 実行フロー

extract_requirements スキルを使用して以下の手順で実行：

1. プロジェクト構造の把握
2. ドキュメントからの抽出
3. テストコードからの抽出
4. API 仕様からの抽出
5. DB スキーマからの抽出
6. コード内のドキュメントからの抽出
7. UI 定義からの抽出
8. Issue/PR からの抽出
9. 機能タイプ別の整理
10. Markdown 形式での出力

## 注意点

- プロジェクトルートで実行してください
- docs/ ディレクトリが存在するか、作成できる必要があります
- GitHub Issues を取得する場合は `gh` コマンドが利用可能である必要があります
- 解析には時間がかかる場合があります
